# ceo — 발동 확인용 스모크 케이스

수치·파일명은 전부 예시(가상)다.

| # | 프롬프트 | 기대 동작 |
|---|---|---|
| 1 | "이 폴더의 보고서 PDF 10건 요약해서 wiki 초안 만들어줘" (위임 언급 없음) | 스킬 자동 발동. "OpenCode에 위임해서 진행합니다" 한 줄 알림 후 `_ceo/01-order.md` 작성 → `opencode run` 실행 위임 → 새 세션 검토 위임 → 판정문 확인 후 보고. Claude가 PDF 원문을 직접 읽으면 실패. |
| 2 | "opencode로 해줘: data 폴더 엑셀 취합해서 월별 집계 DB 만들기" | 트리거 문구로 즉시 발동. 지시서에 입력을 경로로만 지목(내용 복붙 없음), 수용 기준이 개수·필수 항목 등 확인 가능한 조건으로 작성되는지 확인. |
| 3 | "APPROVED랑 REJECTED 판정 기준이 뭐야?" (파일 안 열고 답 가능, 채팅 답변, 30줄 이내) | 위임하지 않고 직접 답변 — 기계적 판정 3조건을 모두 만족하는 유일한 예외 경로. 이걸 위임하면 과잉 발동. |

## 루프 확인 (수동 1회)

검토자가 `VERDICT: REJECTED`를 내도록 수용 기준 하나를 일부러 빡빡하게 잡아 보고:
수정 지시 1~3줄이 order 파일에 덧붙고 `opencode run -c`로 재작업 → 재검토가 도는지,
3라운드 초과 시 사용자 에스컬레이션이 일어나는지 확인한다.
